<h2> ¿Que es git> </h2>
<p> Git es un sistema de control de versiones que permite a los desarrolladores gestionar y rastrear cambios en el código fuente a lo largo del tiempo. </p>
<br>
<h2>¿En que consiste la metolodgia GitFlow?</h2>
<p>GitFlow es una metodología de desarrollo basada en Git que organiza el trabajo en ramas, facilitando la colaboración y la gestión de versiones.</p>
<p></p>Se centra en un modelo de ramificación que incluye ramas principales como master (o main) y develop, así como ramas auxiliares para nuevas características (feature), correcciones de errores (hotfix) y versiones de producción (release).</p> 
<p>Esta estructura permite un flujo de trabajo claro, donde se pueden desarrollar, probar y lanzar nuevas funcionalidades de manera ordenada, minimizando conflictos y asegurando la estabilidad del código en todo momento.</p>
<br>
<h2>¿Porque hemos realizado el proyecto en GitFlow?</h2>
<p>Hemos utilizado GitFlow ya que agiliza y distribuye la carga de trabajo a la hora de hacer proyectos</p>
